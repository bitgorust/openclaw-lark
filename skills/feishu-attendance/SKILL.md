# 飞书考勤

## 执行前必读

- 这组工具以应用身份调用，前提是当前飞书应用已开通考勤相关权限
- 查询排班时，时间跨度不要超过 30 天
- 如果用户只给姓名，没有给员工 ID，先用搜索或通讯录能力把人员标识补齐，再查询考勤
- 考勤结果适合用于“先确认排班，再决定审批/通知/任务安排”，不要跳过核实直接推断

## 核心流程

### 1. 先明确“查排班”还是“查考勤组”

- 目标是某个员工、某批员工某段时间的班次安排：用 `feishu_attendance_shift`
- 目标是某个考勤组本身的配置、负责人、成员范围：用 `feishu_attendance_group`

### 2. 查排班时控制时间窗口

- `check_date_from` 和 `check_date_to` 用自然日表达即可
- 单次窗口不超过 30 天
- 多人多日查询优先聚焦一个业务窗口，不要一次铺太大范围

### 3. 查考勤组成员时按需翻页

- 首屏先取默认页或显式设置 `page_size`
- 返回 `has_more=true` 时，用 `page_token` 继续翻页
- 用户要完整成员清单时继续翻页；只做范围判断时第一页通常够用

## 搜索后的交接

### 排班结果常见后续动作

- 发现排班冲突、请假或加班背景：转到 `feishu-approval`
- 需要提醒值班人或班组成员：转到 IM 发送能力
- 需要把排班结果整理成跟进事项：转到任务工具
- 需要沉淀为说明文档或日报：转到文档工具

### 考勤组结果常见后续动作

- 需要确认成员归属：继续 `list_users`
- 需要联系组内人员：把成员 ID 交给 IM 或搜索能力
- 需要核对某人为什么出现在某个班组：结合排班查询一起看

## 常见错误与排查

| 错误现象 | 根本原因 | 解决方案 |
|---------|---------|---------|
| 排班查不到结果 | 日期格式不对或窗口太大 | 使用自然日格式并控制在 30 天内 |
| 返回权限错误 | 应用未开通考勤相关权限 | 让管理员补齐考勤权限 |
| 考勤组成员不全 | 没有继续翻页 | 检查 `has_more` 和 `page_token` |
| 结果 ID 不适合后续流程 | `employee_type` 选得不对 | 按后续用途选择更合适的 ID 类型 |
