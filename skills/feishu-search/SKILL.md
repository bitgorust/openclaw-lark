# 飞书搜索定位

## 执行前必读

- 不要猜测 `open_id`、`chat_id`、文档 token；用户只给名称或模糊描述时，先搜索再执行后续动作
- 搜索结果可能有重名项；需要结合部门、群成员、文档类型、创建者、时间等信息做二次确认
- 搜索只是定位入口；找到目标后，应立即切换到对应领域工具完成实际操作

---

## 快速索引：目标类型 → 工具

| 目标类型 | 先用哪个工具搜索 | 典型后续动作 |
|---------|------------------|-------------|
| 人员 / 同事 / 审批人 / 参与人 | `feishu_search_user` | 发消息、建任务、建日程 |
| 群聊 / 项目群 / 部门群 | `feishu_chat` (`action: "search"`) | 读消息、发消息、查看群详情/群成员 |
| 文档 / Wiki / 表格 / 多维表格 / 云文档资产 | `feishu_search_doc_wiki` | 读取文档、更新文档、浏览云空间/Wiki、操作 Sheet/Bitable |

---

## 搜索决策

### 1. 目标是“人”时：用 `feishu_search_user`

适用场景：

- “找张三”
- “看看产品经理王敏是谁”
- “给李雷发消息，但我只有名字”
- “把这个任务转给财务同学刘晨”

常用参数：

- `query`：姓名、邮箱、手机号、拼音片段等关键词
- `page_size`：结果较多时扩大返回数
- `page_token`：继续翻页

结果确认重点：

- `name`
- `department_ids` 或部门信息
- `open_id`
- 是否与用户描述的岗位、组织、联系方式一致

找到后如何切换：

- 给人发消息：转到 `feishu-im-send`
- 查与该人的单聊或消息：转到 `feishu-im-read`
- 给该人建任务：使用任务工具/skill
- 邀请该人进日程：使用日历工具/skill

### 2. 目标是“群”时：用 `feishu_chat`

适用场景：

- “找项目 Alpha 群”
- “看看运营周会群”
- “我要往测试群里发通知”
- “群名记不全，只记得包含预算”

第一步先搜索：

```json
{ "action": "search", "query": "项目 Alpha" }
```

必要时再获取详情：

```json
{ "action": "get", "chat_id": "oc_xxx" }
```

结果确认重点：

- 群名称是否匹配
- 是否是目标成员所在群
- 是否需要进一步看群详情或群成员确认

找到后如何切换：

- 读取群历史消息：转到 `feishu-im-read`
- 往群里发消息或回复线程：转到 `feishu-im-send`
- 需要确认群成员：使用 `feishu_chat_members`

### 3. 目标是“文档/Wiki/知识资产”时：用 `feishu_search_doc_wiki`

适用场景：

- “搜一下 OKR 文档”
- “找研发规范 wiki”
- “看看上周编辑过的排班表”
- “找王敏建的那份预算表”

常用搜索方式：

- 关键词搜索：`query`
- 限定类型：`filter.doc_types`
- 限定创建者：`filter.creator_ids`
- 按标题搜：`filter.only_title`
- 按创建/打开时间缩小范围：`filter.create_time`、`filter.open_time`
- 控制排序：`filter.sort_type`

示例：

```json
{
  "action": "search",
  "query": "预算",
  "filter": {
    "doc_types": ["SHEET", "DOCX"],
    "only_title": true
  }
}
```

结果确认重点：

- 文档标题和高亮摘要是否匹配
- 文档类型是否符合目标
- 创建者、最近编辑/打开时间是否符合上下文
- 同名文档较多时，优先通过类型和时间进一步缩小范围

找到后如何切换：

- 读取文档内容：转到文档读取工具/skill
- 更新文档：转到文档更新工具/skill
- 处理评论或媒体：转到 `feishu-doc-collab`
- 浏览文件夹、Wiki 节点、移动复制资产：转到 `feishu-drive-wiki`
- 如果结果是 `SHEET` / `BITABLE`：切到对应表格/多维表格工具

## 搜索后的交接原则

### 人员搜索结果

- `open_id` 是后续发消息、建任务、建日程的关键标识
- 进入单聊消息读取时，可将 `open_id` 交给 IM 读取工具

### 群聊搜索结果

- `chat_id` 是后续读群消息、发群消息、查群成员的关键标识
- 对群目标有歧义时，先 `get` 再继续

### 文档搜索结果

- 后续动作使用搜索结果里的对象标识，不要重新猜测 token
- 文档类型驱动后续工具选择：`DOC`/`DOCX`、`WIKI`、`SHEET`、`BITABLE`、`FILE`、`FOLDER` 各自对应不同能力

---

## 分页与过滤

- `feishu_search_user` 默认返回有限结果；重名较多时应提高 `page_size` 或继续翻页
- `feishu_chat` 搜索到的群较多时，使用 `page_token` 继续取下一页
- `feishu_search_doc_wiki` 最适合配合类型、创建者、时间过滤一起用；只靠关键词时容易出现大量同名结果

---

## 常见错误与排查

| 错误现象 | 根本原因 | 解决方案 |
|---------|---------|---------|
| 直接执行后续动作失败 | 没有先定位真实 ID | 先用搜索工具拿到 `open_id`、`chat_id` 或文档标识 |
| 搜到很多重名人员 | 只用了姓名关键词 | 结合部门、邮箱、手机号或继续翻页确认 |
| 找错群 | 只看群名，没有二次确认 | 对候选群先调用 `feishu_chat.get` 或查看群成员 |
| 文档结果过多 | 没有限定类型/时间/创建者 | 增加 `doc_types`、时间范围或创建者过滤 |
| 找到文档后仍无法操作 | 没按对象类型切换工具 | 根据 `DOCX` / `WIKI` / `SHEET` / `BITABLE` 选择对应工具 |
